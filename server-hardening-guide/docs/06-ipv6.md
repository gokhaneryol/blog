# 6. IPv6 Yönetimi

[← Ana Sayfa](../README.md)

IPv6, modern Linux sistemlerinde varsayılan olarak aktiftir. Kullanılmıyorsa saldırı yüzeyini azaltmak amacıyla devre dışı bırakılabilir. Bu karar bilinçli yapılmalı; özellikle servis sağlayıcının IPv6 gerektirip gerektirmediği kontrol edilmelidir.

## Mevcut Durumu Görme

```bash
# IPv6 adresleri
ip -6 addr

# IPv6 dinleyen servisler
ss -tlnp | grep ":::"

# IPv6 route tablosu
ip -6 route
```

## Seçenek A: Servis Bazlı Kapatma (Önerilen)

Tüm stack'i kapatmak yerine yalnızca ilgili servislerde IPv6 dinlemeyi kapat. Geri alınabilir ve yan etkisi daha azdır.

**BIND9** (`/etc/bind/named.conf.options`):
```bind
listen-on-v6 { none; };
```

**SSH** (`/etc/ssh/sshd_config`):
```ini
AddressFamily inet
```

**Apache** (`/etc/apache2/ports.conf`):
```apache
# Aşağıdaki satırları yoruma al veya kaldır
# Listen [::]:80
# Listen [::]:443
```

**Postfix** (`/etc/postfix/main.cf`):
```ini
inet_protocols = ipv4
```

**OpenVPN** — varsayılan olarak sadece IPv4 tünel açar, ek yapılandırma gerekmez.

## Seçenek B: sysctl ile Tüm Stack'i Kapatma

IPv6'ya hiç ihtiyaç yoksa sistem genelinde kapatmak daha temizdir.

```bash
cat > /etc/sysctl.d/99-disable-ipv6.conf << 'EOF'
net.ipv6.conf.all.disable_ipv6 = 1
net.ipv6.conf.default.disable_ipv6 = 1
net.ipv6.conf.lo.disable_ipv6 = 1
EOF

# Hemen uygula
sysctl --system
```

Doğrulama:
```bash
ip addr | grep inet6
# Çıktı boş olmalı (lo hariç)

cat /proc/sys/net/ipv6/conf/all/disable_ipv6
# 1 döndürmeli
```

Geri almak için:
```bash
rm /etc/sysctl.d/99-disable-ipv6.conf
sysctl --system
```

## UFW ile Koordinasyon

IPv6 stack kapatıldıysa UFW'de de devre dışı bırak:

```bash
sed -i 's/IPV6=yes/IPV6=no/' /etc/default/ufw
ufw reload
```

## Docker ile Dikkat Edilecekler

`sysctl` ile IPv6 kapatıldığında Docker, her konteyner başlatışında container arayüzünde IPv6'yı yeniden etkinleştirmeye çalışabilir. Bu `syslog`'da şu uyarıyı üretir:

```
Foreign process 'dockerd' changed sysctl '/proc/sys/net/ipv6/conf/eth0/disable_ipv6'
from '1' to '0', conflicting with our setting to '1'.
```

Bu kozmetik bir uyarıdır, fonksiyonel soruna yol açmaz. Docker'ın bu davranışını engellemek için `/etc/docker/daemon.json`:

```json
{
  "ipv6": false
}
```

```bash
systemctl restart docker
```
